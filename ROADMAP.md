## 🎯 Project Vision

Provide a **clean, minimal, and modern** React + TypeScript starter that:

- ✅ Requires **zero configuration** to run
- ✅ Demonstrates best practices out of the box (functional components, hooks, strict types, testing)
- ✅ Serves as a **launchpad** for learning, prototyping, or bootstrapping real projects
- ✅ Stays **lightweight** — no extra runtime dependencies beyond React itself

### ✅ v0.1.0 — *Foundation* (Released)
> Initial public release of the starter template.

- [x] Project scaffolded via Create React App
- [x] TypeScript strict typing enabled
- [x] Functional components + React Hooks (`useState`)
- [x] Jest + React Testing Library integration

## 📖 Guiding Principles

| Principle                       | Why it matters                                                                       |
| ------------------------------- | ------------------------------------------------------------------------------------ |
| 🪶 **Stay Minimal**             | The starter should remain easy to read and understand in minutes.                    |
| 👨‍🏫 **Teach by Example**        | Code should reflect modern React + TypeScript best practices.                        |
| 🚫 **Zero Config**              | `npm install && npm start` should be enough to run.                                  |
| 📚 **Documented**               | Every feature and decision should be explained in the README.                        |
| ✅ **Tested**                   | Even a starter should demonstrate how to write meaningful tests.                     |
| 🤝 **Community-Friendly**       | Clear contribution flow, code of conduct, and issue templates.                       |
